# ERD Compiler Architecture

This project is a high-performance Entity-Relationship Diagram (ERD) compiler and visualizer running entirely in the browser. 

## Core Architecture

### State Management
The entire application state is managed by a single Zustand store located in `src/store/useAppStore.ts`.
- **`code`**: The raw string representing the user's DSL input.
- **`tables` & `relationships`**: Parsed representations of the DSL.
- **`nodesLayout` & `edgesLayout`**: Calculated X/Y coordinate layouts generated by `dagre`.

### Layout Engine
The project uses `dagre` (a directed graph layout engine) to calculate the positioning of the schema tables.
- The `compile` method in `useAppStore` passes the tables and relationships to Dagre.
- Dagre calculates the graph, and the results are stored back into the Zustand store as `nodesLayout` and `edgesLayout`.

### Rendering
The visualization is handled primarily by `src/components/DiagramCanvas.tsx`.
- It renders absolute-positioned `<div>` elements for tables based on the `nodesLayout`.
- It renders absolute-positioned `<svg>` `<path>` elements for relationships based on `edgesLayout`.
- The canvas supports panning and zooming (controlled via wheel and drag events).

### Code Editor
The codebase uses `@monaco-editor/react` (in `Sidebar.tsx`) to provide a full VS Code-like editing experience. It registers a custom language `erd-dsl` with custom Monarch tokenizers and Autocomplete item providers to intelligently suggest Lucide icons and hex colors.
